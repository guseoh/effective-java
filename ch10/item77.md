### ✅ 예외를 무시하지 말라

API에서 메소드 선언에 예외를 명시하는 이유는 그 메소드를 호출할 때 발생할 수 있는 예상치 못한 상황(예외)에 대해 **적절한 조치**를 취하거나
필요한 매개변수 등을 **점검해 달라는 경고이자 의무를 부여**하는 것 입니다.

예외를 무시한다는 것은 일반적으로 `try-catch` 문에서 예외를 잡는 `catch` 블록 내에 아무런 코드도 작성하지 않는 것을 의미합니다.  
이는 예외가 발생했음에도 불구하고 오류 상황을 복구하거나 사용자에게 알리거나 로깅하는 등의 필요한 조치를 전혀 취하지 않고 프로그램을 계속 진행하겠다는 뜻입니다.

#### 예외 무시 예시 코드
```java
try{
    //...
} catch (SomeException e){
    
}
```

이렇게 `catch` 블록을 비워두는 것은 예외 상황에 대처하기 위해 존재하는 `try-catch` 문을 작성하는 본래의 목적을 완전히 잃게 만듭니다.

### 번외. 예외를 무시해도 되는 경우(상황)

예외는 기본적으로 무시하면 안 되지만, 아주 가끔 예외를 무시해도 괜찮은 상황이 있습니다.
대표적인 예시가 `FileInputStream`를 닫을 때(close) 발생하는 예외입니다.

`FileInputStream`을 닫을 때는 이미 파일의 상태가 변경된 것이 아니므로, 복구할 대상이 없습니다.
마찬가지로 `FileOutputStream`을 닫는 경우에도 이미 필요한 데이터는 모두 기록된 상태이기 때문에, 남은 작업을 멈출 이유가 없습니다.  
즉, 이 시점에서 발생하는 예외는 ***❗프로그램의 정상 동작에 본질적인 영향을 주지 않는 예외❗*** 입니다.

이런 경우에는 예외를 무시해도 무방하지만 명확한 이유를 남기는 주석과 변수명을 `ignored`로 바꿔 “의도적으로 무시한 것임”을 표현해야 합니다.

```java

public static void main(String[] args) {
    try (FileInputStream input = new FileInputStream("input.txt")) {
        // 파일 읽기 로직 코드
    } catch (IOException e) {
        // 예외 처리 코드
    } finally {
        try {
            input.close();
        } catch (IOException ignored) {
            // 파일 닫기 실패는 복구 불가능하며, 프로그램에 영향 없음
            // 그래서 예외를 무시해도 됨
        }
    }
}

```
